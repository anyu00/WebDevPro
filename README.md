1. **保守性向上** - コードが論理的に分割されている
2. **再利用可能** - Firebase utilities、form logic を他プロジェクトで再利用可能
3. **初心者向け** - 各ファイルの役割が明確
4. **スケーラビリティ** - 新機能追加が容易 (Firebase Auth 追加予定)

## 🚀 機能一覧

### 7つのタブ

1. **カタログ管理** - 新規カタログエントリフォーム
2. **注文する** - 注文フォーム (リッチテキスト対応)
3. **カタログエントリ** - 全カタログをアコーディオンで表示 (インライン編集可)
4. **注文エントリ** - 全注文をアコーディオンで表示
5. **レポート** - 在庫チャート (Chart.js)
6. **カレンダー** - 納品日カレンダー (FullCalendar)
7. **アナリティクス** - 15+ 分析カード (カスタマイズ可, 日付フィルタ付き)

### 📱 モバイル対応

- **900px 以下** → ハンバーガーメニュー (固定サイドバー → スライド式)
- **レスポンシブレイアウト** → カード・フォーム・テーブルが自動調整
- **タッチフレンドリー** → ボタン・フォーム要素がモバイルサイズ対応

### 🔐 Firebase 連携

- **Realtime Database** - Catalogs/ & Orders/ パス
- **CRUD 操作** - 作成・読取・更新・削除すべてに対応
- **リアルタイム更新** - onValue() リスナーで自動同期

## 📖 使い方

### 1. ローカルで実行

```bash
# VS Code で開く
open index.html

# または、Live Server で実行
# VS Code 拡張 "Live Server" をインストール → index.html で右クリック → "Open with Live Server"
```

### 2. カタログを追加

1. 「カタログ管理」タブ
2. フォームに入力 (カタログ名、日付、数量など)
3. "INSERT" ボタンをクリック
4. Firebase に自動保存

### 3. 注文を登録

1. 「注文する」タブ
2. カタログ名・数量・依頼者を入力
3. メッセージ欄でリッチテキスト編集 (太字、斜体など)
4. "注文する" ボタンをクリック

### 4. データを確認・編集

1. 「カタログエントリ」「注文エントリ」タブ
2. テーブルセルをクリックしてインライン編集
3. 削除ボタン で個別削除
4. "サンプルデータ生成" でテスト用データ自動生成

### 5. 分析を確認

1. 「アナリティクス」タブ
2. 期間を選択 (直近7日、30日、90日、またはカスタム)
3. 「カスタマイズ」ボタン で表示するカードを選択
4. リアルタイムでグラフ・数値が更新される

## 🔧 開発者向け情報

### Firebase Config を変更

`js/main.js` の `firebaseConfig` オブジェクトを編集:

```javascript
const firebaseConfig = {
    apiKey: "YOUR_API_KEY",
    authDomain: "your-app.firebaseapp.com",
    databaseURL: "https://your-app-rtdb.firebaseio.com",
    // ...
};
```

### 新機能を追加

例: Google Firebase Auth を統計に追加

1. `js/firebase-config.js` に Auth import を追加
2. `js/main.js` に signIn/signOut ハンドラを追加
3. 認証ユーザーに基づいてデータをフィルタ

### CSS をカスタマイズ

`css/styles.css` を編集:

- **カラースキーム** - `#232946` (紺), `#e0eafc` (薄紫) をカスタマイズ
- **レスポンシブ** - `@media (max-width: 900px)` で調整
- **ガラス効果** - `backdrop-filter: blur()` で半透明調整
